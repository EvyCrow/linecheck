Case 1. 
srand(time(0)) нужно для псевдослучайной генерации чисел на основе времени для заполнения матрицы.
Переменная n задается для генерации матрицы nxn.

Lines 25-35: Ввод n и проверка на то, что n>=0.
Lines 37-39: Задание матрицы nxn.
Lines 41-43: Заполнение матрицы случайными числами (%10 для заполнения однозначными для упрощения восприятия)

Переменные ub, voz, rav, un - счетчики линий по убыванию, по возрастанию, из равных и неупорядоченных соответственно.

Цикл с 47 по 92 строки - проверка строк матрицы.
un2 - костыль, без которого не работала дальнейшая проверка (булевый счетчик в цикле не помогал не засчитывать значение)

Lines 50-62: Проверка строки на убывание (эта и последующие проверки основаны на сравнении первых двух элементов)
Цикл на 52-59 проверяет строку на убывание. В случае, если строка начинает возрастать (равные элементы не считаются за возрастание), счетчик неупорядоченных строк растет.
Lines 60-61: Сравнение изначального значение счетчика неупорядоченных с итоговым. Если различий нет, значит, строка убывает. Соотв. счетчик растет.

Аналогично для строк 64-76 (возрастание) и 78-90 (равенство элементов).

Циклы на 94-99 строках нужны для вывода сгенерированной матрицы (внешний цикл меняет номер строки, внутренний - столбца)
Lines 100-101: Вывод значений счетчиков. Вывод сделан в две строки для удобства чтения кода.
